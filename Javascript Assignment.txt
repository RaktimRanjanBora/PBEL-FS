1.6 Predict the output of console.log(typeof null) and console.log(typeof undefined).
SOLUTION & CODE:
console.log(typeof null);       // "object"
console.log(typeof undefined);  // "undefined"

Explanation: 1. typeof null evaluates to 'object' because of how variables were represented in early JS engines (null was stored as a pointer, sharing the object tag).
2. typeof undefined evaluates to 'undefined' because undefined has its own spec-defined type.
Part 2: Comparison Operators
2.1 Create two variables and check ==, ===, !=, !==, >, <, >=, and <=.
SOLUTION & CODE:
let a = 15;
let b = '15';

console.log("a == b  :", a == b);   // Loose equality
console.log("a === b :", a === b);  // Strict equality
console.log("a != b  :", a != b);
console.log("a !== b :", a !== b);
console.log("a > b   :", a > b);
console.log("a < b   :", a < b);
console.log("a >= b  :", a >= b);
console.log("a <= b  :", a <= b);
CONSOLE OUTPUT:
a == b  : true
a === b : false
a != b  : false
a !== b : true
a > b   : false
a < b   : false
a >= b  : true
a <= b  : true
2.2 Predict the output of console.log(10 == '10') and console.log(10 === '10').
SOLUTION & CODE:
console.log(10 == '10');   // true
console.log(10 === '10');  // false

Explanation: The loose equality operator (==) performs type coercion to match the types before checking value. The strict operator (===) checks both type and value without performing any coercion.
2.3 Predict the output of console.log(null == undefined) and console.log(null === undefined).
SOLUTION & CODE:
console.log(null == undefined);  // true
console.log(null === undefined); // false

Explanation: The JS spec explicitly dictates that loose equality between null and undefined evaluates to true. Under strict equality, they are distinct, separate types and evaluate to false.

Part 4: Arrays

After replacing index 2: [ 'JavaScript', 101, 'Guwahati', null, undefined, 'React' ]

Part 5: Output Prediction
5.1 Predict output for comparison of variables x = 10 and y = '10'.
SOLUTION & CODE:
let x = 10;
let y = '10';
console.log(x == y);  // true
console.log(x === y); // false

Explanation: x == y utilizes coercion and returns true because both represent numeric value ten. x === y checks type natively and returns false.
5.2 Predict output: let arr = [1, 2, 3]; arr.push(4); arr.pop(); console.log(arr);
SOLUTION & CODE:
let arr = [1, 2, 3];
arr.push(4);
arr.pop();
console.log(arr);
CONSOLE OUTPUT:
[1, 2, 3]

Explanation: Push adds 4 to array end. Pop immediately removes the last element (4), leaving the array identical to start.
